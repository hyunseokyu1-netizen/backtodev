## 기본 사용법
### Claude.md 설정 공부
    1. 각 폴더별로 md 파일 생성.
    2. root md 크지 않게 만들기
    3. 반복작업 트리거(예,배포)

### plan edits, accept edits 구분
    1. 어셉트 모드 : 알아서 파일 변경까지해줌
    2. 플랜 모드 : 어떻게 할지만 알려줌.
    3. 플랜모드는 계획만 세우기 때문에 토큰 사용이 적게 들어감. 어셉트 모드는 토큰소모가 크게 들어감. 바로 작업 시작해버리니까…

### ESC 버튼
    1. 작업중 작업 중단. 
    2. esc 두번 누르면 “되돌리기 기능”

### 이미지 올려줄때, 드래그 가능

### iTerm 터미널
    1. cmd+D (패널 세로 분할)
    2. cmd +shift + D (패널 가로분할)
    3. cmd +[ / cmd + ] 패널간 이동

### 터미널 Bash 명령 실행
    1. 프롬프트에서 ! 앞에 붙여서 실행하면 터미널 명령 바로 실행

## 슬래시 명령어
 ### 컨텍스트 관리
        1. /contact , /clear /compact
        2. contact는 토큰 사용량 확인  , 클리어토큰 지우기, 컴팩트는 압축해서 토큰양 줄이기

### 모델 관리 (/models)
        1. opus, sonnet, haiku 
        2. 가장똑똑함, 균형잡힌 성능, 가장 빠름
        3. Pro버젼은 sonnet 만 됨. Opus는 Max 유료 사용 필요
        4. 코딩작업은 opus가 적합, 토큰아끼려면 에이전트 작업시, 서브작업은 sonnet으로 하기

### resume 명령어 
    1. 세션이 닫혔을떄 작업으로 되돌아가기
    2. /resume 으로 돌아가기. 

### mcp 명령어
    1. 내가 사용하는 mcp 서버 
    2. ai가 사용하는 도구라고 생각하기( 무기) 많이 넣을 수록 토큰이 늘어나므로 `커스텀mcp` 추천
    3. 외부도구(Notion, Slack, GitHub) 연결

### 기타
    1. /help ,/config, /export, /output-style 
    2. 도움말, 설정, 채팅보내보기(cursor등으로 옮길때), 응답스타일설정(학습모드, 간결모드 등)
    3. 응답스타일 설정 → 러닝모드는 내가 직접 작성하게끔 해서 배울수있게함.

### 커맨드 만들기
    1. 반복 작업 슬래시 커맨드로 만들어 놓으면 반복작업 / 눌러서 목록에 떠서 실행 가능 함.
    2. 반복작업을 트리커보다 안정적으로 실행할수있음
