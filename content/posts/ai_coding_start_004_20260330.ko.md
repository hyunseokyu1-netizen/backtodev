터미널에서 클로드 코드 사용하다보면, 왔다갔다하면서 네트워크 바뀌면, 세션 끊긴다.

기존에 작업하던 내용들이 날라간다고 생각하면 된다.

그리고 다시 접속해서 하면…. 클로드가 이전에 작업한거는 다 잊고…다시 로그인해주고, 권한 승인해줘야 되는게 많다…

그러다 보니, 그런거 안하려면….세션을 계속 유지해주는게 중요한데.

그거를 도와주는 프로그램이 tmux이다.

vercel 연결이나 github 연결이 끊어져있음.

```jsx
⏺ GitHub 푸시는 정상인데 Vercel 연동 상태를 터미널에서 직접 확인하기 어렵습니다.

  직접 확인하려면:
  1. Vercel 대시보드 → 프로젝트 선택 → Deployments 탭 확인
  2. 또는 ! vercel 명령어로 상태 확인

  몇 가지 가능한 원인:
  - Vercel이 GitHub 레포와 연동은 되어 있지만 자동 배포가 꺼져 있는 경우
  - main 브랜치가 아닌 다른 브랜치가 배포 브랜치로 설정된 경우
  - 빌드 실패로 배포가 중단된 경우
```


그래서 tmux을 설치해 줬다. 

tmux의 장점은 같은 세션에서 여러개 창 만들기와 상태표시줄이 있다는 거다.
