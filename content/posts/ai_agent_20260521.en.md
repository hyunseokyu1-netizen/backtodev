AI Agents are best suited for "say one thing, do several things" tasks. It's best used for tasks that go from research to judgment to execution to validation, rather than just answering questions.

---

## When is it a good fit

Recurring Tasks:** **Recurring Tasks
- Organize blog drafts
- Summarize work based on commit logs
- Write release notes
- Create documentation in the same format every time
- Check images/links/frontmatter

Working with code:** **Working with code
- Find the cause of the bug
- Investigate refactoring scope
- Analyze test failure causes
- Implementing small features
- Consistent fixes across multiple files
- Clean up unused code/dependencies

Research and cleanup:** **Research
- Check usage of specific libraries
- Compare competing services/app features
- Read and summarize documentation
- Analyze project structure
- Track "why this code is the way it is"

Automate your operations:** **Operational automation
- Perform a pre-deployment checklist
- Check SEO metadata
- Check scheduled post publish_date
- Check for broken links
- Update portfolio cards
- Clean up app store description/screenshot wording

---

## When it doesn't fit

- Questions that require a one-line answer
- When the final judgment is more of a matter of taste or strategy
- You need real-time sensitive information and don't have access to it.
- Doing something automatically without authorization that has a high cost of error, such as settlement, deletion, or deployment.
- Things where the requirements are still too vague

In these cases, it's better to set the direction with a human first, and then let the agent execute.

---

## Good use cases

It's a good idea to give agents a combination of "role + input + output + constraints".

Example 1 - Request an investigation:

```
Analyze the portfolio page structure in this repository.
Do not make any modifications,
1. where the data is located
2. where images are loaded
3. what files need to be changed to add a new project card.
to add a new project card.
```

Example 2 - Draft blog:

```
Analyze today's commits and organize them into a blog draft.
The target is today's commits according to git log,
Save the draft in ../blog_doc_temp/.
Stylistically, follow my existing Korean post style.
```

---]

## Practical Standards

| Subject | Role |
|------|------|
| What I'll do | Goals, priorities, and final judgment |
| What the agent will do | Explore, organize, draft, iterate, validate |
| What a human will approve | Deploy, delete, pay, finalize disclosure wording, change sensitive information |

These agents are especially useful for blogging right now.

- blog-auto-draft` - converts today's work into a blog draft
- portfolio-updater` - reads project md and reflects portfolio cards/images
- post-checker` - checks frontmatter, publish_date, links, image paths
- seo-reviewer` - checks title/description/canonical/OG/sitemap
- release-note-writer` - organizes app update history into store copy

The key is less "let the agent do the thinking" and more "let the agent do the tedious and repetitive middle work". It's most reliable to have humans take the final direction and agents focus on speed and miss prevention.

## Create an Agent

Hit `/agents` to select Create New Agent, write the recurring task you want, choose what permissions you want to give it, and it will create it for you.

I created an agent to open my blog draft files, check them, and clean up duplicates.
