# 클로드 Fable 5: 18 일 중단 후 재개, 한국시간 7 월 8 일 오후 3 시 59 분까지 “크레딧 무료” 기간 – 코드 생성 전략까지

## 1. Fable 5 왜 달랐고, 왜 중단됐었나?

Anthropic 의 프런티어 모델인 **Claude Fable 5**는 2026 년 상반기부터 코드 생성, 대규모 코드베이스 분석,跨 모듈 통합 등에서 기존 모델보다 확연히 높은 성능을 보였습니다. 하지만 2026 년 6 월 중반, 미국 정부의 **수출 통제 규정**으로 인해 18 일간 판매 및 접근이 중단되었습니다.

사이버보안 우려로 일부 해킹 프롬프트가 추가로 차단되는 안전 필터가 강화된 상태로, 6 월 30 일 발표 후 **7 월 1 일 전 세계적으로 재개**되었습니다.

## 2. 재개 후 변화: 7 월 1 일 ~ 7 월 7 일 (한국시간 기준)

### 7 월 1 일 재개

- **7 월 1 일 (한국은 7 월 2 일)**부터 Claude.ai 웹/앱과 Claude Code 터미널에서 Fable 5 접근이 전 세계적으로 재개되었습니다.
- Pro, Max, Team, Enterprise 요금제는 **주간 한도 최대 50%까지만** Fable 5 사용이 포함되며, **별도 크레딧 없이** “구독 내 무료”로 쓸 수 있었습니다.

### 7 월 7 일 (한국시간 7 월 8 일) 마감

- 공식 종료 시각: **7 월 7 일 23:59:59 PT (태평양 시간)**.
- 한국시간 (UTC+9) 과 PT(PDT, UTC-7) 차이는 16 시간 →  
  **7 월 8 일 15:59:59 KST (오후 3 시 59 분 59 초)** 까지.
- 이 시간까지는 **Pro 플랜 Weekly limit 50% 내에서 Fable 5를 크레딧 없이** 사용할 수 있습니다.

## 3. 7 월 8 일 이후: 어떤 변화가 있는가?

- **7 월 8 일 (한국시간) 오후 4 시 이후** :  
  Fable 5 사용은 Pro 플랜의 Weekly limit 에 포함되지 않고, **별도 “사용 크레딧 (usage credits)”**을 소모하는 종량제 구조로 전환됩니다.
- **코드 생성 frequent한 사용** :  
  코드 생성은 토큰 소모가 많고 호출이 빈번하므로, 크레딧이 빠르게 소모될 수 있습니다. 실제 비용 부담을 줄이려면 **Fable 5 사용 대상을 엄격히 선별**해야 합니다.

## 4. Pro 플랜 사용자를 위한 Fable 5 코드 생성 전략

### 4.1. 7 월 8 일 오후 3 시 59 분까지: “크레딧 무료” 기간 최적화

1. **무거운 작업에 집중**  
   - Fable 5는 **대규모 코드베이스 분석,跨 모듈 통합, 전체 프로젝트 리팩토링 설계**에 강점이 많습니다.  
   - 이런 작업은 Sonnet/Haiku보다 훨씬 높은 품질로 나올 가능성이 높으므로, **지금만 쓰는 전략**이 합리적입니다.
2. **Sonnet/Haiku로 일상 코딩 처리**  
   - 간단한 스니펫 생성, 문법/스타일 검토, 단위 함수 리팩토링 등은 Sonnet 또는 Haiku로 처리.  
   - Weekly limit 50%를 Fable 5의 “고가의 작업”에 집중하면, 같은 한도 내에서 더 많은 “코드 생성”을 할 수 있습니다.
3. **컨텍스트 최소화**  
   - 프로젝트 구조를 요약한 README, docs 를 먼저 업로드하고, 각 세션에서 정말 필요한 파일만 포함하세요.  
   - 이렇게 하면 Fable 5가 같은 토큰으로 더 많은 “코드 품질”을 뽑아낼 수 있습니다.

### 4.2. 7 월 8 일 이후: 크레딧 소모를 줄이는 방법

1. **Fable 5 사용 시나리오 분리**  
   - **Sonnet**: 일상 코드 생성, 간단한 리팩토링, 테스트 스니펫.  
   - **Fable 5**:  
     - 전체 프로젝트 아키텍처 설계  
     -跨 모듈 리팩토링 전략  
     - 전체 코드베이스 분석을 통한 공통 패턴 발견  
     - 복잡한 디버깅 (예: 여러 모듈이 얽힌 버그)
2. **Plan mode 활용**  
   - Claude Code 의 Plan mode 에서 전체 작업을 단계별로 나누고,  
     - 설계 단계: Fable 5  
     - 파일 단위 코드 생성: Sonnet/Haiku  
   - 이렇게 하면 Fable 5를 “한 번 강력히” 사용하되, 일상 코딩은 크레딧으로 소모하지 않는 구조를 만들 수 있습니다.
3. **크레딧 예산 설정**  
   - 개발 스프린트 기간 (예: 1 주) 을 정하고, 그 기간에 사용할 Fable 5 크레딧 금액을 정해두세요.  
   - “이번 주 Fable 5 크레딧 10 달러”와 같이 제한을 정하면, 무분별한 호출을 막을 수 있습니다.
4. **IDE / 터미널 통합 활용**  
   - Claude Code 를 터미널이나 IDE 에서 직접 사용하면, 프로젝트 전체를 context 으로 보고 짧은 명령어로 여러 파일을 생성/수정할 수 있습니다.  
   - 같은 토큰으로 산출물이 더 많아져, Fable 5 크레딧 소모 대비 효율이 높아집니다.

## 5. 실제 코드 생성 워크플로우 예시 (React Native / Java 프로젝트용)

예를 들어, 기존에 개발 중인 모바일 앱 프로젝트를 기준으로:

1. **프로젝트 아키텍처 설계 (Fable 5)**  
   - 전체 모듈 구조, API 설계, 상태 관리 전략, 네이비게이션 흐름을 문서화.  
   - 이 단계에서 Fable 5를 1 회 강력히 사용.
2. **파일 단위 코드 생성 (Sonnet/Haiku)**  
   - 각 컴포넌트, 유틸 함수, 네이비게이션 설정 등을 Sonnet 으로 쪼개서 생성.  
   - Weekly limit 내에서 충분히 반복 가능.
3. **디버깅 / 리팩토링**  
   - Sonnet 으로 간단한 리팩토링, 오류 수정, 스타일 개선.  
   - 전체 코드베이스를 다시 분석하거나跨 모듈 통합이 필요할 때만 Fable 5 호출.
4. **테스트 코드 작성 (Sonnet/Haiku)**  
   - 테스트 스니펫, 예시 데이터를 Sonnet/Haiku로 생성.  
   - 복잡한 테스트 프레임워크 통합만 Fable 5로.

이런 식으로 **Fable 5는 “무거운” 단계에** 집중하고, **일상적인 코딩은 안정적 모델**로 돌리면, 크레딧 부담을 줄이면서도 코드 생성 효율을 높일 수 있습니다.

## 6. 마무리: 지금 당장 어떻게 쓸 것인가?

- **7 월 8 일 오후 3 시 59 분까지**:  
  - Pro 플랜 Weekly limit 50% 내에서 Fable 5를 크레딧 없이 사용 가능.  
  - 대규모 설계,跨 모듈 통합, 전체 코드베이스 분석 등 “진짜 무거운 작업”에 집중.
- **7 월 8 일 오후 4 시 이후**:  
  - Fable 5 사용은 별도 크레딧 소모.  
  - 일상 코딩은 Sonnet/Haiku로 처리하고, Fable 5는 **필수적인 고가 작업만** 선정해서 사용.

이 전략을 바탕으로, **지금 weeks 안에 진행 중인 프로젝트**에서 Fable 5 사용 분야를 한 번 정리해보시면 좋습니다. 프로젝트 규모, 주요 어려움 (예:跨 모듈 버그, 아키텍처 재설계 필요 등) 을 정리해서, “Fable 5가 꼭 필요한 단계”와 “Sonnet/Haiku로 충분할 단계”를 나누는 것부터 시작하면 됩니다.
