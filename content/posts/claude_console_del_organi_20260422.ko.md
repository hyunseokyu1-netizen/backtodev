## 무슨 일이 일어난 건지 — 콘솔의 계정 구조

사고를 치고 나서야 구조를 이해했다. Anthropic Console은 이런 계층으로 되어 있다:

```
계정(로그인 이메일)
 └── 조직(Organization)   ← 결제, 크레딧, 사용량이 여기 묶임
      ├── API 키
      ├── 결제 정보 / 크레딧
      └── 멤버 관리
```

중요한 건 — **개인 계정도 처음 가입하면 자동으로 '조직'이 하나 만들어지고, 콘솔의 모든 것이 그 조직에 묶인다**는 점이다. 회사 계정에서나 쓰는 개념인 줄 알았는데, 개인 사용자도 예외가 아니다.

그래서 "조직을 삭제하고 새로 만들면 되겠지"는 회사 탈퇴가 아니라 **집을 통째로 철거하는 행위**였다. 조직이 사라지면 소속된 계정은 로그인해도 갈 곳이 없어진다. 그게 내가 본 "로그인이 안 되는" 상태다.

## 결제 버튼이 비활성화되는 흔한 원인들

원래 문제였던 "크레딧 충전 버튼 비활성화"는 나중에 알아보니 조직 삭제까지 갈 문제가 아니었다. 이런 원인들이 흔하다:

1. **결제 프로필 미완성** — 주소, 세금 정보 등 필수 입력란이 하나라도 비어 있으면 버튼이 안 눌린다 (에러 메시지도 없이 조용히)
2. **카드 등록 실패** — 해외 결제 차단된 카드, 3D Secure 인증 실패
3. **브라우저 문제** — 광고 차단 확장이 결제 모듈(Stripe)을 막는 경우. 시크릿 창에서 다시 시도하면 해결되기도 한다
4. **일시적 리전/계정 검증 지연** — 가입 직후에는 결제가 잠시 막혀 있는 경우가 있다

**순서상 "시크릿 창 재시도 → 결제 프로필 재확인 → 지원 문의"로 갔어야 했는데, 나는 곧장 철거반을 불렀다.**

## 같은 상황이라면 — 하지 말 것 / 할 것

| 하지 말 것 | 할 것 |
|-----------|-------|
| 조직 삭제 (= 사실상 계정 초기화) | 시크릿 창 / 다른 브라우저로 재시도 |
| 계정 새로 파서 우회 (지원 복구가 꼬임) | 결제 프로필의 모든 필드 입력 확인 |
| AI 챗봇과 무한 문답 | 챗봇에게 **"조직 복구가 필요하다"고 구체적으로** 말하기 |

마지막 항목이 핵심인데 — 지원 챗봇에게 막연히 "사람 연결해줘"라고 하면 계속 순환 응답만 나온다. 그런데 **"조직을 삭제해서 복구가 필요하다"** 처럼 챗봇이 해결 못 하는 구체적 사안을 말하니 그제야 "사람이 이메일을 보내주겠다"며 에스컬레이션됐다. 챗봇을 통과하는 비밀번호는 '구체성'이다.

## 결말

며칠 뒤 지원팀 이메일로 복구 안내를 받아 해결됐다. 결제 버튼 문제도 결국 결제 정보를 처음부터 다시 입력하니 활성화됐다.
